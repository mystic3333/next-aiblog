Obsidian 有一个公开的秘密：社区插件总数超过 1500 个，但 90% 的用户装了超过 30 个插件后，启动速度开始肉眼可见地变慢，插件之间互相冲突，你甚至不确定某个功能是哪个插件提供的。

这篇文章不是插件推荐清单——那类文章打开率不如直接逛社区。我们要做的事情更具体：用最少的插件搭出最大的能力。

核心逻辑是选三个「枢纽型插件」，其他插件要么不需要，要么只是它们的补充。

## Dataview：淘汰一半笔记

如果你还在手动维护索引笔记、追踪看板、周报汇总，你很可能正在被 Obsidian 的 Markdown 特性欺骗——你在做数据库的工作。

Dataview 把 Obsidian 变成了一个可查询的数据库。你只需要在 frontmatter 里填好结构化数据：

```yaml
---
title: 项目 A 需求评审
status: 进行中
priority: 高
deadline: 2026-07-15
tags:
  - project-a
  - meeting
---
```

然后在另一篇笔记里写：

    ```dataview
    TABLE status, priority, deadline
    FROM #project-a
    SORT priority DESC
    ```

你就不再需要手动维护以下东西了：

- **项目索引笔记**：Dataview 按标签自动生成
- **周报**：按日期范围筛选本周修改的笔记（`WHERE file.mtime >= date(today) - dur(7 days)`）
- **标签汇总页**：一句话查询，按标签分组
- **任务看板**：Dataview 输出 + Kanban 插件的渲染

**正确使用边界**：Dataview 是用来查的，不是用来写的。不要在 Dataview 里做复杂的数据转换逻辑，那是 Templater 的活。

## Templater：消灭重复劳动

Templater 不是「一个模板插件」。它是 Obsidian 的元编程工具。

大多数人的模板文件长这样：

```
---
created: {{date}}
tags: []
---
```

Templater 能做的事情远超这个。三个高阶用法：

**动态模板选择**

按当前光标所在文件夹自动选择不同模板。项目笔记用一个模板，每日笔记用另一个，读书笔记用第三个，全自动：

```
<%* if (tp.file.folder(true) === "Projects") { %>
  <%* tp.file.include("[[模板-项目]]") %>
<%* } else { %>
  <%* tp.file.include("[[模板-通用]]") %>
<%* } %>
```

**自动从文件名提取信息**

文件名 `2026-07-15-项目评审.md` 自动拆解为日期和标题：

```
---
title: <%* tR += tp.file.title.replace(/\d{4}-\d{2}-\d{2}-/, "") %>
date: <%* tR += tp.file.title.match(/\d{4}-\d{2}-\d{2}/)[0] %>
---
```

**批量生成**

QuickAdd + Templater 的组合可以一键生成整组笔记——比如创建项目时同时生成需求文档、技术方案、会议记录三个文件。这个在第四篇展开讲。

## QuickAdd：关闭大部分菜单

Obsidian 的命令面板已经很好了，但你每次建新笔记时，系统流程是：按 `Cmd+N` → 选模板（如果记得的话）→ 放到正确文件夹 → 填 frontmatter → 开始写。

QuickAdd 把它压缩成：按一个快捷键 → 输入标题 → 回车 → 模板自动匹配 → 文件自动归位 → 直接写。

核心配置三件：

- **Capture**：快速捕获想法，自动加入当天的每日笔记，而不打断当前工作流
- **Template**：一键创建带模板的笔记，自动放到正确文件夹
- **Macro**：把多个操作串联成一个快捷键——比如「创建项目笔记 + 同时更新索引文件」

**组合实例**：我只有一个 Obsidian 快捷键绑定——`Cmd+Shift+N` 绑定到 QuickAdd 的通用捕获。其他所有入口都从这一个触发。

## 三件套的协同工作流

这三个插件不是各管一摊。它们互相喂数据：

```
QuickAdd 触发 → Templater 生成笔记结构（含 frontmatter）
              → Dataview 自动查询该笔记，更新所有相关汇总
              → Dataview 输出的内容被 QuickAdd Capture 捕获到当天的临时区域
```

**实际例子**：我在读书读到一段想记的笔记时：

1. `Cmd+Shift+N` → QuickAdd 弹出输入框
2. 输入「心智模型：反脆弱」
3. Templater 自动创建笔记，填好 frontmatter（书名作为标签、当前日期）
4. 笔记自动放在 `Inbox/` 文件夹
5. Dataview 在当周汇总中自动出现这条记录

全程键盘，5 秒，零中断。

## 其他插件要不要装？

按需装，但要遵守一条规则：任何插件如果一个月用不到三次，删掉。

真正值得额外安装的插件很有限：

| 场景 | 插件 | 备注 |
|------|------|------|
| 图表 | Excalidraw | 画架构图/思维导图 |
| 日历 | Calendar | 按日期跳转笔记 |
| 格式增强 | Editing Toolbar | 习惯工具栏的用户 |
| Git 备份 | Obsidian Git | 自动提交到 GitHub |
| 发布 | 见第五篇 | 取决于你的发布方案 |

不需要装的有：任何「All in one」类插件、主题包插件、以及声称「让你的 Obsidian 更好看」的十几个 CSS 片段插件。

## 一句话

枢纽型插件解决 80% 的问题，剩下 20% 按需填补。装得少才能用得深。
